# 画面詳細設計

このディレクトリには、各画面の詳細設計を個別ファイルとして管理します。

## 画面一覧

| 画面名 | 画面ID | ファイル名 | 説明 |
|-------|--------|----------|------|
| メイン画面 | SCREEN-001 | main-screen.md | URL入力、調査開始 |
| 調査結果一覧画面 | SCREEN-002 | results-list-screen.md | リンク切れの一覧表示 |
| 調査結果詳細画面 | SCREEN-003 | result-detail-screen.md | リンク切れの詳細情報表示 |
| 設定画面 | SCREEN-004 | settings-screen.md | 調査オプション設定 |
| 調査履歴画面 | SCREEN-005 | history-screen.md | 過去の調査履歴一覧 |

## 画面遷移図

```
メイン画面
  ├─→ 調査開始 → 調査結果一覧画面
  │                    ├─→ 調査結果詳細画面（詳細表示）
  │                    └─→ メイン画面（戻る）
  ├─→ 設定画面
  │     └─→ メイン画面（戻る）
  └─→ 調査履歴画面
        ├─→ 調査結果一覧画面（履歴選択）
        └─→ メイン画面（戻る）
```

## UIコンポーネント設計

### 共通コンポーネント
- **Button**: ボタンコンポーネント
- **Input**: 入力フィールドコンポーネント
- **Table**: テーブルコンポーネント
- **ProgressBar**: プログレスバーコンポーネント
- **Modal**: モーダルダイアログコンポーネント
- **Navigation**: ナビゲーションコンポーネント

### 専用コンポーネント
- **URLInput**: URL入力コンポーネント（メイン画面）
- **ResultTable**: 調査結果テーブルコンポーネント（調査結果一覧画面）
- **ResultDetail**: 調査結果詳細コンポーネント（調査結果詳細画面）
- **SettingsForm**: 設定フォームコンポーネント（設定画面）
- **HistoryList**: 履歴一覧コンポーネント（調査履歴画面）

## レスポンシブデザイン詳細

### ブレークポイント
- デスクトップアプリケーションのため、固定サイズのウィンドウ
- ウィンドウサイズのリサイズは可能（最小サイズ: 800x600）

### 各デバイスでの表示
- Windowsデスクトップアプリケーションのため、デスクトップ表示のみ

## ファイル命名規則

画面ファイルの命名規則：
- 画面名ベース: `main-screen.md`, `results-list-screen.md`
- 機能ベース: `settings-screen.md`, `history-screen.md`

推奨: 画面名と機能を表す分かりやすい名前を使用してください。
