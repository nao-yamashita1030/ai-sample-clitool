# 画面詳細設計: 調査履歴画面

## 基本情報

- **画面名**: 調査履歴画面
- **画面ID**: SCREEN-005
- **URL**: `/history`
- **説明**: 過去の調査履歴一覧を表示する画面
- **対象ユーザー**: すべてのユーザー

## 画面の目的

- 過去の調査履歴を一覧表示する
- 履歴を選択して調査結果を表示する

## レイアウト

### 画面構成

```
┌─────────────────────────────────────────┐
│           ヘッダー（戻るボタン）          │
├─────────────────────────────────────────┤
│                                         │
│  ┌─────────────────────────────────┐  │
│  │  履歴一覧テーブル                │  │
│  │  [テーブルヘッダー]              │  │
│  │  [履歴行]                        │  │
│  │  [ページネーション]               │  │
│  └─────────────────────────────────┘  │
│                                         │
│  ┌─────────────────────────────────┐  │
│  │  アクションボタン                │  │
│  │  [戻る]                         │  │
│  └─────────────────────────────────┘  │
│                                         │
└─────────────────────────────────────────┘
```

## 画面要素

### 要素1: 履歴一覧テーブル

- **要素名**: 履歴一覧テーブル
- **種類**: テーブル
- **説明**: 調査履歴をテーブル形式で表示
- **カラム**: 
  - 調査URL
  - 調査日時
  - ステータス
  - 総ページ数
  - リンク切れ数
  - 詳細ボタン

## 画面遷移

### 遷移元

- [メイン画面](./main-screen.md) → この画面（調査履歴リンククリック）

### 遷移先

- この画面 → [調査結果一覧画面](./results-list-screen.md)（詳細ボタンクリック）
- この画面 → [メイン画面](./main-screen.md)（戻るボタンクリック）

## 処理フロー

1. 画面表示時
   - `ipc:get-scan-history` APIを呼び出し
   - 履歴一覧を表示
2. 詳細ボタンクリック時
   - 調査結果一覧画面に遷移（scanHistoryIdを渡す）
